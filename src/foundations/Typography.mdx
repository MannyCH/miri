# Typography

Typography establishes visual hierarchy and readability in the Miri meal planning application.

Miri uses **24 text styles** across 6 categories, combining two font families:
- **Baloo Bhaijaan 2** - Friendly, rounded font for display and headings
- **IBM Plex Sans** - Clean, professional font for body text and UI elements

---

## Font Families

```css
--font-family-display: 'Baloo Bhaijaan 2', system-ui, -apple-system, sans-serif;
--font-family-heading: 'Baloo Bhaijaan 2', system-ui, -apple-system, sans-serif;
--font-family-body: 'IBM Plex Sans', system-ui, -apple-system, sans-serif;
--font-family-tiny: 'Inter', system-ui, -apple-system, sans-serif;
```

## Typography Scale Summary

| Style | Font Family | Size | Line Height | Weight | Use Case |
|-------|-------------|------|-------------|--------|----------|
| Display | Baloo Bhaijaan 2 | 56px | 64px | Regular/SemiBold | Hero sections |
| Heading 1 | Baloo Bhaijaan 2 | 40px | 48px | Regular/SemiBold | Page titles |
| Heading 2 | Baloo Bhaijaan 2 | 32px | 40px | Regular/SemiBold | Section titles |
| Heading 3 | Baloo Bhaijaan 2 | 24px | 32px | Regular/SemiBold | Subsections |
| Heading 4 | Baloo Bhaijaan 2 | 20px | 28px | Regular/SemiBold | Card titles |
| Body | IBM Plex Sans | 16px | 24px | Regular/SemiBold | Main content |
| Body Small | IBM Plex Sans | 14px | 20px | Regular/SemiBold | Secondary content |
| Annotation | IBM Plex Sans | 14px | 150% | Regular/SemiBold | Metadata |
| Tiny | IBM Plex Sans | 11px | auto | Regular/SemiBold | Dense UI |
| Uppercase | IBM Plex Sans | 14px | 24px | SemiBold | Labels |

## Best Practices

‚úÖ **DO:** Use heading styles in hierarchical order (H1 ‚Üí H2 ‚Üí H3)  
‚úÖ **DO:** Combine typography classes with semantic color tokens  
‚úÖ **DO:** Use body text for main content, body small for secondary  
‚úÖ **DO:** Maintain consistent line-height for readability  

‚ùå **DON'T:** Skip heading levels (e.g., H1 ‚Üí H3)  
‚ùå **DON'T:** Use tiny text for critical information  
‚ùå **DON'T:** Hardcode font sizes or line heights  
‚ùå **DON'T:** Mix too many font weights in one component
